What is Tidus?

Your AI requests, always on the optimal model

Tidus sits in front of your AI workloads as a self-hosted FastAPI service. You call one endpoint — Tidus picks the right model automatically.

Most teams waste money sending every request to the same premium model. A simple classification task costs $15/1M tokens on Claude Opus when Gemini 2.0 Flash at $0.10/1M returns an equivalent result.

Tidus analyses each request's complexity, privacy requirements, capability needs, and budget constraints — then scores every eligible model using a weighted algorithm (70% cost, 20% tier, 10% latency) to find the optimal route.

Because Tidus is fully self-hosted and open source, no request data ever leaves your environment. It works with any AI vendor and supports local Ollama models for confidential workloads where cloud APIs are prohibited.

The weekly pricing registry keeps costs accurate without manual intervention — syncing prices from multiple sources every Sunday, detecting outliers with MAD-based consensus, and creating versioned, auditable revisions.

tidus_client.py
# Before Tidus — every call goes to GPT-4o
response = openai.chat.completions.create(
  model="gpt-4o",  # $2.50/1M — always
  messages=messages,
)

# After Tidus — optimal model selected per request
response = await tidus.route(
  messages=messages,
  complexity="simple",
  team_id="analytics",
)
# → Gemini 2.0 Flash at $0.10/1M ✓ 96% cheaper

# Critical task? Tidus routes to Tier 1 automatically
response = await tidus.route(
  messages=messages,
  complexity="critical",
)
# → Claude Opus 4.6 ✓ best model for the job

Five Pillars

Everything routing needs to work in production

Tidus v1.1.0 delivers five integrated capabilities — from real-time routing to weekly market intelligence.

Pillar 1
🧭
Intelligent Model Routing
A 5-factor scoring algorithm evaluates every eligible model on cost (70%), tier (20%), and latency (10%). Complexity tiers (simple → critical) define which model tiers are eligible. Capability matching filters models by required features — multimodal, code, function-calling. Budget guardrails cut off models that would exceed team spend limits.
MAD scoring Capability matching Tier ceiling Privacy routing Local-first
Pillar 2
🏛️
Self-Healing Pricing Registry
Prices drive 70% of the routing score — stale prices cause systematic misspending. The registry syncs from multiple sources every Sunday using MAD-based outlier detection. Changes create versioned, audited DB revisions via atomic two-phase write. Rollback to any previous revision in one API call. Overrides let teams adjust pricing, disable models, or pin providers without touching config files.
Multi-source consensus MAD outlier detection Atomic revisions 90-day audit trail RBAC overrides
Pillar 3
🛡️
Budget Guardrails
Set hard spend limits per team, per model, or globally. Every routed request records actual token usage and cost in real time. Overages trigger alerts and optionally fall back to cheaper models within the same tier. Billing reconciliation ingests provider CSV exports and flags mismatches between Tidus cost records and actual invoices — catching silent cost leakage before it compounds.
Per-team limits Real-time enforcement Overage alerting CSV reconciliation
Pillar 4
🔬
Drift Detection & Auto-Remediation
AI models change behaviour without notice — latency spikes, context limits shift, tokenization diverges, prices move. Tidus runs four drift detectors on a 5-minute cycle: Latency (measured P50 vs catalog baseline), Context (overflow rate in recent requests), Tokenization (estimated vs actual token divergence over 7 days), and Price (unusual change frequency or magnitude). Warning events escalate the model to priority probing. Critical events auto-create a hard_disable override and remove the model from routing immediately. After 3 consecutive healthy probe cycles, the override is automatically lifted and the model returns to rotation. Every event is linked to the exact registry revision that was active at detection time — enabling precise postmortems.
4 drift detectors 5-min probe cycle Auto-disable Auto-recovery Revision-linked events Prometheus metrics
Pillar 5
📰
Weekly Market Intelligence
Every Sunday at 02:00 UTC, Tidus syncs prices and emails your team a market intelligence report. The magazine highlights price drops (savings opportunities), price increases (cost risk), new model additions, and models with stale data. Delivered as a richly-formatted HTML email — no third-party analytics, no tracking pixels.

How It Works

From request to optimal route in milliseconds

Tidus evaluates every request through a deterministic pipeline — no magic, no black-box AI decisions.

Step 1
📥
Receive Request
FastAPI endpoint accepts messages, complexity hint, capability tags, team ID, and optional privacy flag
Step 2
🔍
Filter Candidates
Capability matching removes ineligible models. Tier ceiling from complexity. Privacy flag forces local-only. Budget check removes over-limit models
Step 3
💰
Estimate Cost
Token count × registry price per model. 15% safety buffer applied. Prices from the active revision — verified against multiple sources
Step 4
📊
Score & Rank
score = 0.70 × cost_score + 0.20 × tier_score + 0.10 × latency_score. Lowest cost wins within each tier
Step 5
⚡
Route & Record
Top-ranked model receives the request. Actual tokens and cost recorded to cost_records. Telemetry updates latency P50
